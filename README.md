# 고장 길잡이 (demo-fix-guide)

현장 AS 기사를 위한 이슈 대응 가이드 앱. 증상을 검색하면 원인 후보와 단계별 조치를 바로 보여준다.

- **증상 검색 + 계통별 브라우즈** — 전원·구동·온열·소음·외관 5개 계통, 이슈 30건
- 이슈마다 **원인 후보 → 조치 단계(체크리스트) → 에스컬레이션 기준**
- **내 해결 메모**와 해결 기록 — 데이터는 내 폰에만 저장 (회사 보안 걱정 없음)
- **오프라인 동작** — 지하 매장·기계실 등 통신이 안 되는 현장에서도 전체 가이드 사용 가능

> 이 앱은 **"회사원도 이런 앱을 만들 수 있다"**를 보여주는 교육용 데모 시리즈의 하나입니다.
> 담긴 이슈 데이터는 **가상의 안마의자 제품** 기준으로 AI가 생성한 것입니다 — 여러분 회사의
> 매뉴얼·AS 일지를 넣으면 여러분 팀의 실전 도구가 됩니다.

---

## 🙋 이 앱, 나도 만들 수 있을까?

**네, 코딩을 몰라도 됩니다.** 이 앱은 AI(Claude)에게 **말로 시켜서** 만든 것입니다. 핵심은 두 가지였습니다: ① 앱의 화면과 동작을 구체적으로 설명하기, ② 이슈 데이터(증상→원인→조치)를 AI에게 **미리 생성**시키기.

### 준비물

1. **Claude** — [claude.ai](https://claude.ai) (웹) 또는 Claude Code(터미널)
2. **우리 회사의 이슈 지식** — AS 매뉴얼, 자주 받는 문의, 선배 기사의 노하우 메모 등. 없으면 AI에게 초안을 만들게 해도 됩니다 (이 데모가 그렇게 만들어졌습니다)

### 따라 만들기 — 이 프롬프트를 그대로 써보세요

```
AS 기사가 현장에서 쓰는 이슈 대응 가이드 웹앱을 만들어줘.

- 우리 제품은 [제품명]. 계통을 전원/구동/온열/소음/외관 5개로 나눠줘
- 증상 키워드로 검색하면 해당 이슈가 나오고,
  이슈를 열면 원인 후보 → 단계별 조치(체크하며 진행) → 언제 본사에
  넘겨야 하는지(에스컬레이션 기준)가 보인다
- 이슈별로 내 메모를 남길 수 있고, "해결 완료"를 누르면 기록에 쌓인다
- 데이터는 서버 없이 브라우저에만 저장, 오프라인에서도 동작(PWA)
- 어두운 배경에 큰 글씨, 모바일 우선

그리고 [제품명]의 흔한 고장 증상 30개를 계통별로 만들어서
증상/원인/조치/에스컬레이션 구조의 데이터로 넣어줘.
```

**포인트**: AI 답변 기능을 앱에 넣는 게 아니라, **AI에게 이슈 데이터를 미리 만들게 해서** 앱에 담습니다. 그래서 API 키도, 사용료도, 인터넷도 필요 없는 앱이 됩니다. 실제 도입할 땐 AI가 만든 초안을 베테랑 기사가 검수하는 흐름을 권합니다.

### 더 쉬운 방법 — 이 저장소를 통째로 보여주기

Claude Code에서 이 저장소를 열고:

```
src/data/issues.js의 구조를 유지하면서, 내용을 우리 회사 [제품명]의
이슈로 교체해줘. 첨부한 AS 매뉴얼(PDF)을 참고해.
```

## 📚 알아두면 좋은 개념 (비개발자용 최소 지식)

| 개념 | 한 줄 설명 |
|---|---|
| **웹앱** | 설치 없이 브라우저 주소로 여는 앱. 앱스토어 심사 없이 URL로 배포 |
| **PWA** | "홈 화면에 추가"하면 진짜 앱처럼 아이콘·전체화면·**오프라인 실행**이 되는 웹앱 |
| **localStorage** | 서버 없이 **내 기기 안에** 데이터를 저장 — 회사 보안 제약을 피하는 열쇠 |
| **빌드타임 AI** | AI를 앱 실행 중에 부르지 않고, **만들 때만** 써서 데이터·이미지를 미리 생성하는 방식 |
| **저장소(리포지토리)** | 코드와 변경 이력을 담는 폴더. GitHub는 이를 보관·공유하는 서비스 |
| **배포** | 내 컴퓨터의 앱을 인터넷 주소로 여는 것. 이 시리즈는 Vercel(무료)을 씁니다 |

## 🔍 더 알고 싶을 때 조사하는 법

1. **Claude에게 바로 물어보기** — "PWA가 뭐야? 초등학생에게 설명하듯"처럼 눈높이를 지정하세요.
2. **막히면 에러를 그대로 붙여넣기** — 해석하려 하지 말고 복사해서 "이거 왜 이래?"
3. **공식 문서는 AI와 함께 읽기** — URL을 주고 "요약해줘, 내 상황엔 뭘 쓰면 돼?"
4. **이 저장소의 [CLAUDE.md](CLAUDE.md)** — 앱 구조와 동작 원리 정리. AI에게 읽혀도 됩니다.
